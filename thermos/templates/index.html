This course will teach you to use pip and virtualenv for cleaner and efficient package-management while developing web Applications with python & flask framework. Concepts Like:

MVC Model-Template-View pattern in Flask, Or as it is more commonly known as the Model-View-Controller (MVC-architecture)

    Model

    Software components to:

  • Hold data
  • Usually represents rows in a database (routes of data in dbs)
  • Flask leaves it to developers for writing these components
  • sqlite3, SQLALchemy (are popular choices, but others can be used as well)
    • *SQLAlchemy was used for this project.

    Template

    Known as view in MVC terms, these software components are used to:

  • Generate HTML output for our webpages.
  • Flask includes Jinja2
    • *Jinja2 was used for this project.

    View

    Known as Controller in MVC terms, these software components are used to:

  • Generate Responses to a specific wrb request.
  • Mapped to one or more URLs
Templates & Views
    Flask comes with a Template Engine Jinja 2, That can generate any kind of text base file format, Not just HTML. But in Flask project, it's main use will be to generate HTML.

    Jinja 2

    HTML files & Jinja2 templates to:

  • Generate HTML from data that we pass from Views Functions
  • Jinja2 concepts of Maintainable links, Conditional logic, looping, Macros, Super Block, Template Inheritance were used through the overall course.

    • Views

    Handles all aspects of :

  • Routing data to specific URLs
  • Passing Variables/Lists/Objects(instances of Class) to Jinja2 Template Engine
    • Little info on running app in debugging mode.
